# pangea-ring0-os (Version 0.0.1)

Welcome to **pangea-ring0-os**！ 
本プロジェクトは、Rust言語の持つ「強力な型システム」「所有権構造」「ゼロコスト抽象化」を極限まで活かし、ハードウェアの直上（リング0）で動作する次世代のベアメタル（実機・エミュレータ直上のローレイヤ）オペレーティングシステムを構築する研究開発プロジェクトです。

バージョン `0.0.1` では、開発環境の完全な固定化（Nixによる決定論的なビルド環境の構築）と、ベアメタルRustツールチェーンの確立、そしてQEMUエミュレータ上での安全な起動基盤の設計を達成しています。

OSのアーキテクチャ（MulticsやUNIX、Plan 9、L4マイクロカーネルなどの歴史的知見）をリスペクトしつつ、現代のサイバーセキュリティにおける「攻撃者の視点」を先回りした防御的設計（Offensive Defense）を組み込んでいくための第一歩となるリポジトリです。

---

## 🛠 プロジェクトの特徴

1. **メモリ安全なリング0（カーネル空間）の実装**
   Rustの厳格なコンパイル時チェックにより、従来のC/C++製カーネルで多発していた「バッファオーバーフロー」や「データレース（多核種間での競合）」を未定義動作（UB）になる前に物理的に排除します。
2. **Nixによる環境の一元管理（Determinism）**
   「開発者によってコンパイラのバージョンが違う」「QEMUやGDBのバージョンが異なって動かない」という、OS開発で最も苦痛となるトラブルをNix Flakesによって完全に解消しています。
3. **LLVM/LLDリンカの強制適用**
   GNU ldではなく、Rustと親和性の高いLLDリンカを強制適用することで、リンク時間の劇的な短縮と、カーネルバイナリのメモリフットプリントを極限まで削ぎ落とすLTO（Link Time Optimization）の基盤を整えています。
4. **ハッカーマインドに裏打ちされた防御設計**
   ページテーブルによるメモリ空間の隔離、IDT（中断・例外ハンドラ）の厳格なカプセル化、コンテキストスイッチ時のレジスタ漏洩防止など、脆弱性を悪用させない堅牢なハードウェア・ソフトウェアの境界線（Boundary）を意識した設計を行います。

## 📂 ディレクトリ構造（Directory Layout）

OS開発では、どのファイルがどこに配置されているかを正確に把握することが重要です。本プロジェクトは以下の清潔な階層構造（ディレクトリ表示）を採用しています。

```text
pangea-ring0-os/
├── .cargo/
│   └── config.toml         <- Rustのビルドターゲットやコンパイラフラグを指定する重要設定
├── src/
│   └── main.rs             <- カーネルのエントリポイント（#![no_std] および #![no_main]）
├── flake.nix               <- モダンなNixシステムのための環境定義ファイル
├── shell.nix               <- 従来の nix-shell やCI環境のためのスタンドアローン環境定義ファイル
├── rust-toolchain.toml     <- 使用するRustコンパイラ（Nightly）とコンポーネントの厳密な指定
└── README.md               <- 本ドキュメント

⚙ 開発環境の構築（Setup Environment）

本プロジェクトは、ホストOS（UbuntuやNixOSなど）のグローバル環境を一切汚染しないように設計されています。
前提条件

ホストマシンに Nixパッケージマネージャ がインストールされている必要があります。また、Flakes機能が有効になっていることを確認してください。
🚀 開発環境への入り方
パターンA：モダンな Nix Flakes を使用する場合（推奨）

プロジェクトのルートディレクトリで以下のコマンドを実行します。
Bash

nix develop

パターンB：従来の nix-shell コマンドを使用する場合

Flakesが使えない環境やCI（継続的インテグレーション）環境では、以下のコマンドで自動的に同一の隔離環境が構築されます。
Bash

nix-shell

    ⚠️ 超重要トラブルシューティング
    Nixは安全のため、Gitの管理下にないファイルをビルドコンテキストから隔離します。新しく作成したファイル（rust-toolchain.toml など）がNixに認識されない場合は、必ず以下のコマンドを叩いてGitのインデックス（ステージング領域）に登録してください（GitHubにPushしたりCommitしたりする必要はありません）。
    Bash

    git init (まだリポジトリを初期化していない場合)
    git add flake.nix shell.nix rust-toolchain.toml src/main.rs .cargo/config.toml

🏃‍♂️ ビルドと実行方法（Build & Run）

Nixの開発シェルに入った状態で、以下のコマンドを実行します。
1. カーネルバイナリのビルド
Bash

cargo build --target x86_64-unknown-none

コンパイルが成功すると、target/x86_64-unknown-none/debug/pangea-ring0-os に純粋な静的リンクバイナリ（ELF形式）が生成されます。
2. ヘルパースクリプトによる実行

Nix環境内に自動インジェクションされた cargo-qemu コマンドを使用することで、将来的にブートイメージ化からQEMUの立ち上げまでを1発で自動実行できるようになります。
Bash

cargo-qemu
